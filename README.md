具体的なjavacdocコメントを作成してください。
- Javaソースコード名
:TimeRecorder.java
- クラス名
:public class TimeRecorder
- クラスの目的:出退勤を管理する
- 利用方法:
1. コマンドライン引数で出勤と退勤を区別する
    - "1"が出勤で、"0"が退勤とする
2. 実行すると標準出力に以下の情報を出力して終了する
    - 出勤 yyyy/mm/dd hh:mm:ss
    - 出力の様式は以下の通り
        - 出勤時は"出勤"、退勤時は"退勤"を出力し、1文字の空欄を開ける
        - "yyyy/mm/dd hh:mm:ss"は現在日時

3. 追加機能
    - 出退勤をファイルに記録する
        - ファイル名: time_record.dat
        - 形式: テキストデータ（UTF-8）
        - ファイルが存在する場合は末尾に追記する。存在しない場合は作成する
    - 標準出力とファイルに同じ様式で出力する

4. チェックポイント03の詳細設計（追加機能
    - 操作ログを記録する
       - ファイル名: timerecorder.log
       - 形式: テキストデータ（UTF-8）
       - ファイルが存在する場合は末尾に追記する。存在しない場合は作成する
    - 記録内容および書式は以下の通り
        - コマンドライン引数
            - "yyyy/mm/dd hh:mm:ss [INFO] コマンドライン引数"
        - 出退勤の出力結果
            - "yyyy/mm/dd hh:mm:ss [INFO] 出退勤の出力結果"
        - エラーメッセージ
            - "yyyy/mm/dd hh:mm:ss [ERROR] エラーメッセージ"
        - 例外発生時の出力結果
            - "yyyy/mm/dd hh:mm:ss [CRITICAL] 例外メッセージ"

5. チェックポイント04の詳細設計（仕様変更）
    - クライアントアプリとサーバーアプリに分ける
        - アプリ間の通信方法: java.net.Socket, java.net.ServerSocketを用いた TCP/IP通信
        - クライアント: TimeRecorder.java
            - コマンドライン引数: 変更なし
            - 実行時の処理:
                - 出退勤情報を標準出力へ出力する（出力様式を含めて変更なし）
                - 出退勤情報をサーバーへ送信する（追加）
                    - サーバーへ出力するのは出勤/退勤の情報のみ
                    - 時刻はサーバー側の時計を用いる
                - 出退勤記録ファイルへの出力はオミットする
                - アプリを終了する
        - サーバー: TimeRecordServer.java
            - 実行すると、受信待ち状態になる
            - 受信した場合の動作:
                - 出退勤情報を標準出力と出退勤記録ファイルへ出力する
                - 出退勤情報の出力様式はTimeRecorderと同じ
                - 受信待ち状態に戻る
